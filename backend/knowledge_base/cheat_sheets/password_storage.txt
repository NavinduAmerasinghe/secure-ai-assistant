Password Storage Cheat Sheet

Key Secure Coding Rules:
- Never store plain text passwords.
- Use strong, salted password hashing algorithms (e.g., bcrypt, Argon2).
- Enforce password complexity and rotation policies.

What Developers Should Avoid:
- Storing passwords in plain text or reversible encryption.
- Using outdated hash functions (MD5, SHA1).

Safer Alternatives:
- Use password hashing libraries designed for security.

Remediation Guidance:
- Audit password storage and migrate to secure hashing if needed.
